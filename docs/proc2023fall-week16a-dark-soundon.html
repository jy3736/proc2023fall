Enough JavaScript for Node-RED

Variables, data types

JavaScript is a loosely typed programming language, meaning variables can hold values of any data type without requiring explicit declarations upfront. Some primary data types include:

Numbers - Integers like 10 or floating point values like 3.14159. Useful for math calculations. Strings - Represent textual data defined within single or double quote marks. Let you work with words and characters. Booleans - True/false logical values, great for conditional tests. Undefined - Variables declared but not assigned hold the undefined value. Useful for validating if variables have been initialized. Null - Represents the absence of a value. Helpfully allows explicitly clearing object properties or pointers.

You define variables using the var, let or const keywords:

let counter = 10;
const apiKey = '123MY_KEY';

Use const for values that won't change and let for variables you reassign.

A variable declared anywhere in a JavaScript function is available throughout the function. Functions provide scope - variables inside aren’t available globally.

As a dynamic language, the same JavaScript variable can be reassigned to hold different data types over time:

let myVar = "Hello";
myVar = 10;
myVar = true;

This flexibility is convenient but can also lead to unintended coercion, so take care to know the type being stored at any given time.

Functions, arguments

Functions provide reusable blocks of code that can be called from different parts of a program to perform a specific task.

Functions take input via arguments, process the data, and usually return an output value or object.

For example:

function doubleNumber(num) {
  return num * 2;
}

doubleNumber(5); // Returns 10

Defining functions in JavaScript involves:

  • The function keyword
  • A meaningful imperative name
  • Parameters provided inside parentheses
  • The function body contained in { } braces
  • Using return to return values

Function names should use camelCase and clearly indicate what they do.

You can call defined functions elsewhere in your code by name and passed argument values will be assigned to the listed parameters.

Arguments get passed by value in JavaScript, meaning a function receives copies of the values rather than direct references. This allows modifying arguments inside functions without impacting the original variables.

However, objects get passed by reference, allowing directly mutating object properties inside functions. Understanding this distinction takes practice.

Well structured functions accept necessary inputs, execute a single focused task, and avoid shared dependencies to maximize reusability.

Conditional logic

Conditionals provide a way to selectively run certain blocks of code based on the evaluation of expressions that resolve to true or false.

The primary conditional in JavaScript is the if statement:

let temp = 22;

if (temp > 30) {
   console.log("Hot outside!");
} else if (temp < 16) {
   console.log("Chilly day!");
} else {
   console.log("Moderate temperature.");
}

Breaking this down:

  • The if statement evaluates the expression inside parentheses and executes the following code block if true
  • else if clauses let you chain additional tests
  • else provides a default case if none match

Some key notes on conditional syntax:

  • Code blocks are wrapped in { } braces
  • Comparators include >, <, <=, >= to compare values
  • == loose equality checks and === strict equality compare without type coercion
  • Multiple conditions can be chained using && || logical operators

The ternary operator provides a shortcut syntax for if/else expressions:

let canVote = (age >= 18) ? true : false;

With the condition before ?, the true expression after ?, and false expression after the : colon.

Conditionals are fundamental building blocks in JavaScript. Expressions usually check a variable’s value to decide which path to execute. Complex conditionals might be abstracted into reusable functions or excellent use cases for switch statements.

Loops

Loops allow repeating execution of a section of code multiple times without needing to rewrite repetitive logic. JavaScript includes several loop construct options:

for Loop

for (let i = 0; i < 5; i++) {
  console.log("Iterating!", i);
}
  • Commonly used classic loop structure
  • Great for iterating a known number of times
  • Use a counter variable (i) that gets updated each pass
  • Initialize, evaluate condition, increment in one line

while Loop

let input = getInput();

while (!input) {
  input = getInput(); // Fetch user input
}
  • Code executes over and over as long as condition is true
  • Useful when exit condition is event triggered

do while Loop

let input = getInput();

do {
  // Process input
} while (!input);
  • Similar to while but runs at least once before checking test
  • Avoid needing separate first check

Can also iterate arrays or objects:

let people = [{name: "John"}, {name: "Mike"}];

people.forEach(person => {
  console.log(person.name);
});
  • Concise way to execute logic for each item
  • Arrow func arguments provide access

Break and continue keywords allow finer grain control over loop execution.

Getting practice iterating data structures and objects with different looping approaches accelerates mastery.

Working with objects

Objects represent real-world entities comprised of a collection of related properties and methods. For example a car object:

let car = {
  make: "Toyota",
  model: "Camry",
  year: 2022,
  wheels: 4,
  drive: function() {
    console.log("Vroom!");
  }
};

Objects provide a natural way to model composite real-world things.

Defining objects uses { } brace notation with comma-separated key:value pairs inside:

  • Keys act as property identifiers
  • Values can contain data types or functions
  • Javascript object properties can also point to other nested objects

Access values on an object instance using dot or square bracket notation:

car.make; // "Toyota"
car['model']; // "Camry"

Dot notation simpler for known valid identifiers. Bracket access allows dynamic property names.

Functions attached directly on objects are called methods - these define object capabilities:

person.walk();
circle.getArea();

The this keyword available in methods refers to the parent object instance.

JavaScript classes provide an alternate concise syntax for defining object structure reusable across instances.

Handling errors

Errors refer to runtime problems that cause code to fail. In JavaScript, errors immediately halt execution when thrown.

Robust applications anticipate and handle errors gracefully to avoid application crashes. For example:

try {
  nonExistentFunction();
} catch (error) {
  console.log("Application recovered from error: " + error);
}
  • The try block wraps around logic that may throw exceptions
  • catch block executes only in case of errors
  • Error variable contains info about what went wrong

Can handle multiple specific error cases:

try {

} catch (TypeError) {

} catch (ServerError) {

}

Specific error info helps correct issues.

You can also create custom application error classes extending the built-in Error class:

class InvalidInputError extends Error {
  constructor(message) {
    super(message);
  }
}

throw new InvalidInputError("Invalid input");

This allows throwing application-specific errors to better isolate issues without exposing internals through generic JavaScript errors.

Other language features like strict mode, linting and debugging stacks also help harden code against potential issues.

Writing resilient code requires an eye towards anticipating failures - with good practice this becomes second nature.
