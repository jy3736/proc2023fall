Overview of Node-RED Visual Programming Concept

Node-RED provides a browser-based flow editor that allows developers to wire together flows using a wide range of nodes. It follows a visual programming paradigm - rather than writing code, you create programs by connecting nodes graphically.

The editor provides an easy to use interface. You simply drag and drop nodes from the palette onto the canvas and connect them with wires to build complete applications. The flows can take in data from various sources, process it, and output results to a range of services.

Node-RED builds on top of Node.js and uses its event-driven, non-blocking model to enable building scalable network applications. The flows created in Node-RED execute in a Node.js runtime.

It comes packaged with a library of built-in nodes that can be used to quickly create flows. You get support for HTTP, WebSocket, TCP, MQTT and more right out of the box. Many open source contributed nodes are also available to install from the Node-RED library for connecting to third-party systems.

Core Nodes - Inject, Function, Debug

Some key built-in nodes that form the core building blocks in Node-RED flows are:

  • Inject node - generates test data to stimulate a flow
  • Debug node - used to display messages flowing through a flow for debugging
  • Function node - allows writing custom JavaScript functions to implement application logic

Flows, Wires, Input & Output

The core model of Node-RED consists of flows, nodes, and messages:

  • Flows - act as container structures housing a group of connected nodes
  • Nodes - represent a piece of code that does something. Have defined inputs and outputs.
  • Messages - get passed between nodes along wires as inputs and outputs

Clicking on wires lets you view the message payload that gets passed between nodes. This makes debugging flows very intuitive in Node-RED.

Example Simple Flows

Here are some simple example flows you can create quickly in Node-RED to demonstrate its visual programming model:

  • Inject timestamp -> Debug output
  • HTTP Input -> Function -> HTTP Response
  • WebSocket listener -> Process data -> Database storage
  • Email trigger -> Create PDF -> Email out

The flow editor makes it very easy to get started wiring up simple building blocks even without writing any code!
